# Manual Changes

Files that required manual edits beyond what was auto-generated by the connector tooling.

## Java Files

- `src/main/java/io/ballerina/stdlib/mi/executor/BalExecutor.java`
  - Added `restoreNativeMessage()` to reconstruct the Message BMap from the persisted native `ServiceBusReceivedMessage` and set it in args before settlement function invocation
  - Added `constructMessageRecord()` to build a Ballerina Message record from the native message fields
  - Updated `persistNativeMessage()` to store native messages in a map keyed by sequence number, with support for both single Message and MessageBatch results

- `src/main/java/io/ballerina/stdlib/mi/Constants.java`
  - Added `ASB_NATIVE_MESSAGE` constant for persisting native messages in the Synapse message context

## UI Schema Files

- `src/main/resources/uischema/messagereceiver_Receivepayload.json`
  - Added `T` (Output Type) combo box field with options `json`, `xml`, `string` (default: `json`) in the Input Variables group. Auto-generated tooling produces a plain `stringOrExpression` input; the combo constrains the value to the three supported typedesc literals.

- `src/main/resources/uischema/messagesender_Send.json`
  - Changed `validateType` of `applicationProperties_properties` from `""` to `"regex"` with `matchPattern: "^$|^(\\{[\\s\\S]*\\}|\\$\\{.+\\})$"`. This enforces that the field is either empty, a JSON object literal `{...}`, or a dynamic expression `${...}`, preventing plain strings from being passed as application properties.
