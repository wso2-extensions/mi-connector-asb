# Manual Changes

Files that required manual edits beyond what was auto-generated by the connector tooling.

## Java Files

- `src/main/java/io/ballerina/stdlib/mi/executor/BalExecutor.java`
  - Added `restoreNativeMessage()` to reconstruct the Message BMap from the persisted native `ServiceBusReceivedMessage` and set it in args before settlement function invocation
  - Added `constructMessageRecord()` to build a Ballerina Message record from the native message fields
  - Updated `persistNativeMessage()` to store native messages in a map keyed by sequence number, with support for both single Message and MessageBatch results
  - Added `buildMessageReturnType()` to programmatically construct a typed `Message` record (with `json`, `text`, or `xml` body type) using `TypeCreator`, based on the user-selected `returnType` template parameter
  - Modified `invokeMethodSync()` to accept an optional `Type returnType` and use the typed overload of `invokeMethodAsync` for type-guided body binding
  - Added logic in `execute()` to look up the `returnType` template parameter and inject the correct `BTypedesc` for the `T` parameter by matching `paramType{i}=typedesc` and `param{i}=T` in the operation context before invocation
  - Added `responseBodyType` propagation through `execute()` to `processResponse()` to control output serialization format
  - Updated `processResponse()` to serialize the result as XML (`bMapToOMElement`) or JSON (`bMapToJsonObject`) based on `responseBodyType`
  - Replaced `stringValue()`-based BMap serialization with `bMapToJsonObject()` and `bValueToJsonElement()` which use Gson directly, fixing incorrect escaping of string field values containing special characters
  - Added `bMapToOMElement()` to serialize a Message record to XML when `returnType` is `xml`

- `src/main/java/io/ballerina/stdlib/mi/Constants.java`
  - Added `ASB_NATIVE_MESSAGE` constant for persisting native messages in the Synapse message context
  - Added `TEXT` constant for the `text` return type option

## UI Schema Files

- `src/main/resources/uischema/messagereceiver_Receive.json`
  - Added `returnType` (Return Type) combo field with options `json`, `text`, `xml` in the Input Variables group. Controls how the message body is bound when receiving.

- `src/main/resources/uischema/messagereceiver_Receivepayload.json`
  - Added `T` (Output Type) combo box field with options `json`, `xml`, `string` (default: `json`) in the Input Variables group. Auto-generated tooling produces a plain `stringOrExpression` input; the combo constrains the value to the three supported typedesc literals.
  - Updated `help` text to note that this operation can only be used in Receive and Delete mode and is not compatible with Peek Lock mode.

- `src/main/resources/uischema/messagesender_Send.json`
  - Changed `validateType` of `applicationProperties_properties` from `""` to `"regex"` with `matchPattern: "^$|^(\\{[\\s\\S]*\\}|\\$\\{.+\\})$"`. This enforces that the field is either empty, a JSON object literal `{...}`, or a dynamic expression `${...}`, preventing plain strings from being passed as application properties.

- `src/main/resources/uischema/messagesender_Sendpayload.json`
  - Updated `help` text to note that the payload is sent as `application/octet-stream` to Azure Service Bus.
